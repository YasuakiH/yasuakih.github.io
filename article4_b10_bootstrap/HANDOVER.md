## 1. このコードが解く課題

品質工学・信頼性工学を実践する品質管理担当者が直面する「早期信頼性把握」および「予防保守ジレンマ」の解決策を検討する。

### 問題の構造

```
【現実の状況】
  新型機を導入した直後 → 稼働データが少ない
  ↓
  突然死を避けるため予防保守(PM)で部品を定期交換する → 打ち切りデータ(Censored)が発生
  ↓
  打ち切りにより真の摩耗故障寿命が見えない → 「新型は旧型と同等か?」が判断できない

【このコードの回答】
  Kaplan-Meier法 (KM法)でCensoredを正しく処理 + ブートストラップでB10の不確実性を可視化
  → 「現時点での確信度」つきで同等性を判定する
```

---

## 2. 設計思想

### 思想①: Censored データを捨てない

`bootstrap_b10_local()` は KM 法を使い、打ち切りデータを生存率推定の**分母**に組み込む。
`compute_km_simple()` と `compute_km_with_ci()` の両 KM 関数は、この思想の実装である。
**Failed データだけで百分位点を計算するコードに書き換えてはならない。**

### 思想②: 点推定の危険性 → 不確実性の可視化

B10 ライフを「1つの値」で報告するのではなく、ブートストラップ分布として表現する。
グラフ⑦の「差の信頼区間 vs 許容限界」がこの思想の核心である。

### 思想③: TOST (Two One-Sided Tests) による同等性判定

```
差 D = B10(新型) - B10(旧型) の90%信頼区間[CI_lower, CI_upper] を算出。

主判定: CI_lower >= 許容下限 (旧型B10 × EQUIV_MARGIN_LOWER_RATIO)
        → True なら「同等以上」
```

上限 (`EQUIV_MARGIN_UPPER_RATIO`) は参考表示のみで判定には使わない。
これは「新型が旧型以上であること」を主目的としているため。

## 4. 変更してはならないコード

### `bootstrap_b10_local()` のガード処理

```python
# ① Failed 3件未満チェック
if int(np.sum(events)) < 3:
    return np.array([])

# ② β の発散防止
if slope <= 0.3:
    continue

# ③ オーバーフロー防止
if b10_x >= 20:
    continue
```

これら3つのガードを削除・緩和すると、`inf` や数値爆発が発生する。

### `WEIBULL_MODES` の変更影響範囲

`WEIBULL_MODES` を変更すると以下の2箇所に影響する。

| 影響箇所 | 影響内容 |
|---|---|
| `generate_fleet_data()` | 生成されるフリートデータの寿命分布が変わる |
| `plot_ax3_bathtub_curve()` | バスタブ曲線の形状・混合比が変わる |

③のバスタブ曲線は `generate_fleet_data()` と**同じseed=42**で乱数を再現し、`scaling_factor` を復元している。シードを変える場合は両方を揃える。

## 5. 改造パターン

### パターンA: 評価月数を変えてシミュレートする
```python
EVAL_MONTHS_NEW = 24  # ← ここだけ変える (6, 12, 24, 36, 48, 60, 72)
```
他は何も変えなくて良い。全グラフが自動的に対応する。

### パターンB: 許容マージンを変える
```python
EQUIV_MARGIN_LOWER_RATIO = -0.15  # -10% → -15% に変更
```
`plot_ax7_equivalence_test()` の緑帯と判定ロジックが自動で更新される。

### パターンC: グラフを追加する
GridSpec は `3×3` で固定されている。グラフを追加する場合:
1. `height_ratios` や `GridSpec(3, 4, ...)` など全体レイアウトを変更する
2. `main()` 内の `gs[行, 列]` の割り当てを見直す
3. 新しい `plot_axN_...()` 関数を追加する (既存関数を壊さない)

### パターンD: 新型機のパラメータを変える
```python
B10_DESIGN_NEW = 120_000  # 設計ライフを変更
CENSORING_FACTOR = 6.0    # PM間隔を長くする
```
`PM_CYCLES_NEW` は自動計算されるため個別に変更不要。

## 8. 信頼性工学の用語解説

| 用語 | 意味 |
|---|---|
| B10ライフ | 累積故障率が10%になる稼働サイクル数。部品寿命の標準指標 |
| Censored | 故障前に部品を交換したデータ。「打ち切り」とも呼ぶ |
| KM法 | Kaplan-Meier法。Censoredを正しく扱う生存率推定法 |
| ワイブル分布 | 故障寿命モデルに広く使われる確率分布。β(形状)とη(スケール)で定義 |
| β<1 | 初期故障型(右下がりハザード)。初期不良・製造ばらつき |
| β=1 | 偶発故障型(水平ハザード)。ランダム外乱・偶発的破損 |
| β>1 | 摩耗故障型(右上がりハザード)。疲労・摩耗・老化 |
| TOST | Two One-Sided Tests。同等性を「差の信頼区間と許容限界の比較」で判定する手法 |
| ブートストラップ | 復元抽出の反復により推定量の分布を経験的に得る手法 |
