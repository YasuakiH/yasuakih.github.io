## 1. このコードが解く問題

### 出発点となった現場の問い

生産用機械の品質管理を志す者が、新型機を導入した後に直面する本質的な問いがある。

> **「新型機の部品ライフが旧型機と同等以上であることを、早期に客観的に確認したい。
> 何ヶ月のデータを集め、何件の故障を観測すれば、判断が下せるか?」**

新型機は稼働台数が少なく、その信頼性も高く、特に初期には故障データは極めて乏しい。ワイブルプロットで有意な評価を行うには一般的に10件以上の故障データが必要とされるが、それが揃うまで半年〜1年以上かかることもある。この「早期評価」の難しさを解決するために本シミュレーターを作成した。

### 技術的な障壁: 予防保守による打ち切り

現場には追加の難しさがある。予防保守(PM)によって部品は故障前に交換され、その稼働サイクルは打ち切りデータ (Censored) になる。
Censored を無視して故障データだけで B10 を計算すると、B10推定値は過小評価される方向にバイアスを受ける。こうした課題に対応するために Kaplan-Meier法 (KM法) を採用する。

### このシミュレーターの回答

```
【問い】
  何ヶ月・何件あれば客観的に判断できるか?

【回答の構造】
  Weibull モデルで現実を再現する (データ生成)
          ↓
  市場の稼働実績と運用実態を把握する (①〜⑤)
          ↓
  KM 法で Censored を処理 → Weibull 回帰で B10 を推定する (⑥⑦)
          ↓
  ブートストラップで「いつ判断できるか」を可視化する (⑧)
          ↓
  ワイブルプロットで推定結果を目視確認する (⑨)
```

## 2. 設計思想 (変更前に必ず確認)

### 思想①: Weibull モデルを共通言語とする

このシミュレーターは Weibull モデルが最初から最後まで貫く設計とした。

| フェーズ | Weibull の役割 | 対応グラフ |
|---|---|---|
| データ生成 | 混合ワイブル分布でバスタブ型の部品寿命を生成 | ①バスタブ曲線 |
| B10 推定 | KM 推定点を Weibull 確率紙変換し β・B10 を逆算 | ⑦⑧ |
| 結果確認 | ワイブルプロットで回帰直線と B10 交点を目視 | ⑨ |

これにより、信頼性工学でワイブル分析を学んだ者が、「バスタブ型モデルを起点に、Weibull プロットで締めくくる」という一本のストーリーを描けるようにした。

### 思想②: Censored データを捨てない - KM 法による橋渡し

`bootstrap_b10_local()` は KM 法を使用して、打ち切りデータを生存率推定の分母に組み込む。これによって打ち切り (Censored) と故障 (Failed) の両方を含む寿命データを Weibull 回帰へとつなげる。

```
KM で生存曲線を推定
    ↓
Weibull 確率紙変換: X = ln(t), Y = ln(-ln(S(t)))
    ↓
初期故障領域 (F < 30%) で線形回帰 → 傾き = β, 切片から η を復元
    ↓
B10 に対応する Y 値 = ln(-ln(0.9)) を代入して B10 を逆算
```

`compute_km_simple()` と `compute_km_with_ci()` はこの橋渡しの実装である。Failed データだけで百分位点を計算するコードに書き換えてはならない。

### 思想③: 点推定でなく不確実性を可視化する

B10 ライフを「1つの値」で報告するのではなく、ブートストラップ分布として表現する。グラフ⑦の「差の信頼区間 vs 許容限界」とグラフ⑧の「信頼区間の収束」がこの思想の核心となる。

### 思想④: 統計的根拠で同等性を判定する - TOST

```
差 D = B10(新型) − B10(旧型) の90%信頼区間 [CI_lower, CI_upper] を算出。

主判定: CI_lower ≥ 許容下限 (旧型B10 × EQUIV_MARGIN_LOWER_RATIO)
        → True なら「同等以上」
```

上限 (`EQUIV_MARGIN_UPPER_RATIO`) は参考表示のみで判定には使わない。
旧型にもブートストラップを適用してペアワイズ差を構築することで、旧型の推定不確実性も判定に反映させている。

## 4. 変更してはならないコード (触れる前に必ず読む)

### `bootstrap_b10_local()` のガード処理

```python
# ① Failed 3件未満チェック
if int(np.sum(events)) < 3:
    return np.array([])

# ② β の発散防止 (Weibull 回帰の数値安定性)
if slope <= 0.3:
    continue

# ③ オーバーフロー防止
if b10_x >= 20:
    continue
```

これら3つのガードを削除・緩和すると `inf` や数値爆発が発生する。

### `WEIBULL_MODES` の変更影響範囲

`WEIBULL_MODES` を変更すると以下の2箇所に影響する:

| 影響箇所 | 影響内容 |
|---|---|
| `generate_fleet_data()` | 生成されるフリートデータの寿命分布が変わる |
| `plot_ax3_bathtub_curve()` (グラフ①) | バスタブ曲線の形状・混合比が変わる |

①のバスタブ曲線は `generate_fleet_data()` と同じseed=42で乱数を再現し、`scaling_factor` を復元している。seedを変える場合は両方を揃えること。

## 5. よくある改造パターンと注意点

### パターンA: 評価月数を変えてシミュレートする
```python
EVAL_MONTHS_NEW = 24  # ← ここだけ変える (6, 12, 24, 36, 48, 60, 72)
```
他は何も変えなくて良い。全グラフが自動的に対応する。

### パターンB: 許容マージンを変える
```python
EQUIV_MARGIN_LOWER_RATIO = -0.15  # -10% → -15% に変更
```
グラフ⑦の緑帯と TOST 判定ロジックが自動で更新される。

### パターンC: グラフを追加する
GridSpec は `3×3` で固定されている。グラフを追加する場合:
1. `height_ratios` や `GridSpec(3, 4, ...)` など全体レイアウトを変更する
2. `main()` の `gs[行, 列]` の割り当てを見直す
3. 新しい描画関数を追加する (既存関数を壊さない)

**グラフ①のバスタブ曲線は `gs[0, 0]` (上段左端) に固定すること。
ストーリーの起点であり、最初に目に入る位置に置く設計意図がある。**

### パターンD: 新型機のパラメータを変える
```python
B10_DESIGN_NEW = 120000  # 設計ライフを変更
CENSORING_FACTOR = 6.0    # PM間隔を長くする
```
`PM_CYCLES_NEW` は自動計算されるため個別変更不要。

## 8. 信頼性工学の用語解説

| 用語 | 意味 |
|---|---|
| B10ライフ | 累積故障率が10%になる稼働サイクル数。部品寿命の現場標準指標 |
| Censored | 故障前に部品を交換したデータ。「打ち切り」とも呼ぶ |
| ワイブル分布 | 故障寿命モデルに広く使われる確率分布。β(形状)とη(スケール)で定義 |
| β < 1 | 初期故障型 (右下がりハザード)。初期不良・製造ばらつき |
| β = 1 | 偶発故障型 (水平ハザード)。ランダム外乱・偶発的破損 |
| β > 1 | 摩耗故障型 (右上がりハザード)。疲労・摩耗・老化 |
| バスタブ曲線 | 3つの故障モードを混合したハザード関数の模式図 |
| KM法 | Kaplan-Meier法。Censoredを正しく扱う生存率推定法。Weibull回帰への橋渡し |
| ブートストラップ | 復元抽出の反復により推定量の分布を経験的に得る手法 |
| TOST | Two One-Sided Tests。同等性を「差の信頼区間と許容限界の比較」で判定する手法 |
