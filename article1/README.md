# 【制作中】モンテカルロ法によるデジタル印刷機の顧客未知パラメータ推定

この記事は、デジタル印刷機を用いてオンデマンド印刷を営む印刷業者が製造している印刷物の特徴を、コンピュータ・シミュレーションによって推定するものである。

## 目的
デジタル印刷機の保守サービスを最適化するプロセスをコンピュータ・シミュレーションで実行する。プロセス全体を3つの独立したテーマに分割して、この記事ではその最初のステップを説明する。汎用のプログラミング言語Pythonとフリーソフトでシミュレーションを構築し、架空の印刷機の未知パラメータを推定する。

* 1.顧客の未知パラメータ推定 【本記事の範囲】
* 2.部品ライフ推定
* 3.機械の信頼度成長

## 環境と構成要素
### デジタル印刷機
デジタル印刷機は、印刷物を生産財として必要な部数だけ生産するオンデマンド印刷に用いられる高速・高性能な印刷装置である。印刷機はその性能を維持するためにドラム、ブレード、ベルトなどの消耗品を定期的に交換する必要がある。メーカーは、定期交換部品ごとに寿命の管理目標を設定し、計画的に交換することで印刷機の故障を未然に防ぐ保守サービスを提供している。高い信頼性が求められる印刷機は、通信ネットワークを介して稼働状況を送信する機能を持つ。メーカーは、印刷業者 (顧客) の送られた稼働状況を用いて機械の状態を把握することができる。

### 定期交換部品と部品ライフ
印刷機の性能を維持するために定期的に交換される部品を定期交換部品という。これらの部品は使用によって消耗するので、一定の基準に基づいて交換を要する。部品の消耗度合いを部品ライフといい、例えばその部品で印刷した用紙の枚数数や、部品の運動した回数や距離のような指標で測定される。メーカーは、機械の設計時に部品ライフの管理目標を策定し、保守サービスではこれを基準に部品を交換する。

### 遠隔保守で収集される情報
印刷機から遠隔保守のために送信される情報は、印刷機が消費したインクや用紙の量である。遠隔保守では、これらに基づき消耗品の消耗度合いを推定し、次の交換時期を推定する。

* インク総量: 印刷機が消費したインクの量
* 用紙サイズ分布: 印刷ジョブに含まれる用紙サイズの分布
* 印刷ページ長分布: 印刷ジョブに含まれる印刷ページ長の分布

### 予防交換と故障交換
定期交換部品の交換では印刷機を止める必要があるため、印刷業者の業務計画に沿うことが望ましい。複数の保守作業を同時にするなど計画的にすることで機械の稼働率を維持できる。計画的な保守では、部品ライフの推定値が管理目標に到達する前であっても交換する「予防交換」が行われる。これに対して突発的な故障が起こって稼働中の機械が止まった場合は、回復が急がれるため、問題を起こした部品のみ交換する「故障交換」とよばれる保守形態がなされる。

予防保守を顧客の業務に合わせて最適化するために顧客固有の情報が必要であるが、通常、顧客情報保護の観点から得られないため)、利用可能な情報から推定する必要がある。

### 印刷ジョブ
印刷業者が1つ1の印刷物を製造する工程を印刷ジョブと呼び、次のようなパラメータの集まりである。印刷ジョブは印刷業者のビジネス成績につながるために秘匿性も高く、1つ1つは分からない。

* トータルエリアカバレッジ: 用紙面積に対するインクが塗布された面積の割合 (TAC値, エリアカバレッジとも)。印刷物の原稿によって決定される。
* 印刷ページ長: 印刷ジョブに含まれるページ数。複数部数の場合は印刷部数だけ同じ内容が繰り返される。
* 用紙サイズ: 印刷ジョブで指定される用紙の大きさ (例: はがき、A5、A4、B4、A3ノビ、長尺)
* 両面比率: 印刷物全体に対する両面印刷の比率。(両面印刷/(両面印刷+片面印刷))

カラー印刷物の場合は、色に関するパラメータも必要となる。ただし、この記事では簡略化のためにモノクロとした。(カラー比率は0%)

* 色指定: 使用するインクの種類
* カラー比率: 印刷物全体に対するカラー印刷の比率。(カラー印刷/(カラー印刷+モノクロ印刷))

### 未知パラメータ
定期交換部品の部品ライフは印刷機にかかる負荷に依存する。部品の交換を最適化するためには、顧客がどのように印刷機を使用し、どれだけの負荷が部品にかかるかを推定する必要がある。顧客の使い方を特徴づける外部から観察できない情報を「未知パラメータ」と呼ぶ。例えば、印刷業者が受注した印刷物の仕様のようなビジネス情報 (例: 用途、寸法、用紙、色指定、加工、部数) や、それを制作するために印刷機へ指示する印刷ジョブ、また、印刷業者が顧客から預かった印刷原稿自体もまた未知パラメータとなる。

## 未知パラメータの推定方法

### 考え方
解析的に解くための数式が不明であることから、モンテカルロ法で解を探索する。未知パラメータを仮定し、それに基づく印刷ジョブを多数発生させる。それぞれのジョブで印刷機に入出力される資源を計算・累積したものと、印刷機の稼働実績とを比較する。両者の違いが小さければ設定した未知パラメータは「もっともらしい」と見なすことができる。

#### 推定したい未知パラメータ
部品ライフを算出する観点では、関心のある部品にかかる負荷を表す指標が未知パラメータである。これらは個々の印刷物によって異なるために統計分布で表される。

- インクの物理量
  - トータルエリアカバレッジ (用紙の単位面積あたりのインク塗布量)
- 印刷用紙の物理量
  - ジョブ長 (印刷ジョブに含まれる総ページ数。書籍の場合、ページ数 x 部数)
  - 用紙サイズ (ページあたりの紙の面積、あるいは部品の回転数や移動距離)

顧客の品質基準 (例: 不良とみなす画質不具合の種類や程度) に対する関心は、個々の印刷機での部品ライフに向けられる。

#### 印刷ジョブと資源量との関係
印刷ジョブと、対応する資源量 (インク、用紙) との関係は次の式で表わされる。インク原単位は、単位面積あたりに塗布されるインクの質量で、印刷機の機種ごとに定められる定数である。印刷ジョブ長はジョブに含まれるページ数であり、1部あたりのページ数が部数分だけ繰り返され、用紙に印刷 (面付け) される時の両面/片面の指示によって実際に使われる用紙数が決まる。

- インク量
  - インク量/ページ [g/page] = 用紙サイズ[m2/page] x トータルエリアカバレッジ[-] x インク原単位 [g/m2]
  - インク量/印刷ジョブ [g/job] = ページ毎インク消費量[g/page] x 印刷ジョブ長[page/job]
- 用紙量
  - 用紙枚数/印刷ジョブ [paper/job] = 印刷ジョブ長[page/job] x 両面印刷比率[-]

#### 印刷機の稼働実績
保守サービスのために印刷機から稼働情報が、通常は集約された統計量として得られる。シミュレーションでは、仮定した未知パラメータに基づいて推定した統計量と比較することで、その仮説の妥当性を評価する。

- 資源
  - 総インク消費量
  - 総用紙消費量
- 用紙サイズ分布
- 印刷ジョブ長分布

#### 収束の可否
パラメータの組み合わせは無数にあるが、デジタル印刷機で制作する印刷物にはパターンが存在する。たとえば教科書のように文字が主体であれば、用紙サイズがB5～A4で、エリアカバレッジは低いと考えられる。カラフルなカタログであれば用紙サイズはA4～長尺で、エリアカバレッジも高い。こうしたパターンに当てはまるパラメータの組み合わせは前述の制約を満たして抽出され、それ以外は除かれると考えられる。

## シミュレーションの設計
### Step1. 印刷ジョブのモデリング
#### 未知パラメータの仮定 (潜在変数)
前述のとおり、未知パラメータとして、インクの物理量と、用紙の物理量を用いる。シミュレーションでは、2つのパラメータの大きさを同時かつ直感的に把握できるよう、2つのパラメータを軸とする二次元のマップ上の位置に対応づけた。これを「印刷機セグメント」と呼ぶ。またシミュレーションを簡単にするために、それぞれのパラメータを3段階 (Lo, Hi, Mid) の離散的な値とした。これにより二次元マップはを9つの領域 (3 x 3) に分けられ、図のように Q1-Q9 のアドレスを割り当てた。1つのアドレスの指定と、2つのパラメータの指定は同等である。シミュレーションでは未知パラメータを仮定するためにアドレスQ1-Q9を無作為に選択した。

##### 印刷機セグメント (TACや印刷ジョブ長の分布の仮説)
印刷機セグメントは、印刷機の運用形態を二次元マップ上の位置で代表させる概念である。それぞれの軸は、トータルエリアカバレッジの軸と、印刷ジョブ長の軸である。シミュレーションでは、印刷機セグメントを無作為に選択し、これによってトータルエリアカバレッジと、印刷ジョブ長とを決定する。

### コマンドライン
``` shell
usage: sim_hidden_param.py [-h] [--iterations ITERATIONS] [--printing_machines [PRINTING_MACHINES ...]] [--pickle PICKLE] [--cpu_count CPU_COUNT] [--seed SEED] [--import_file IMPORT_FILE]

options:
  -h, --help            show this help message and exit
  --iterations ITERATIONS
                        モンテカルロ法の総実行回数を指定する (デフォルト: 1)。 実用上の上限は 1000 程度。 (例: --iterations 1000)
  --printing_machines [PRINTING_MACHINES ...]
                        シミュレーション対象の印刷機の名前 (例: --printing_machines PM1 PM2)
  --pickle PICKLE       既存のシミュレーション結果 pickle ファイルを指定する。この場合、シミュレーションは行わなずに、シミュレーション結果だけを表示する。大域変数 sim_result_all に格納されたデータはデバッグ時に生かせる。マルチプロセスの場合は機能しない制約がある。(例: --pickle )
  --cpu_count CPU_COUNT
                        シミュレーションをマルチプロセスで行う場合、使用するCPU数を指定する。1=シングルプロセス(デフォルト), 2以上=マルチプロセス)
  --seed SEED           random.seed() を指定する。0 の場合はシステム時刻を使う (デフォルト: 42)。 (例: --seed 0)
  --import_file IMPORT_FILE
                        印刷機のデータファイル
```

### 実行例

``` shell
python sim_hidden_param.py --iterations 100 --printing_machines PM1
```
