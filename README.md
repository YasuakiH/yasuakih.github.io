# [定期交換部品の交換時期の最適化に関するコンピュータシミュレーション](https://yasuakih.github.io/)

ビジネス、特に修理可能な製品をもつメーカーでは、顧客先に保守作業員を派遣し、消耗品 (定期交換部品) を定期的に交換するなど、製品の安定稼働と性能維持のための保守サービスを提供している。こうした保守サービスを計画的に運用するためには、消耗品が所定の寿命を果たすことが前提となる。もし、寿命が計画よりも短ければ製品は予期せず故障し、それを使用していた顧客は業務に影響を受けるだろう。他方メーカーも、保守作業員を緊急に派遣して故障を修復し、機能を回復する責を負わなければならない。そこで信頼性を重視した製品では、計画外の保守発生を防ぎ、顧客業務の停止を避けるために、稼働状況をメーカーに通知する仕組み (遠隔保守) を備えている。

定期交換部品の交換時期を決めるために、その寿命を推定する必要がある。しかし寿命には、顧客の用途や品質基準など、さまざまな要因 (隠れパラメータ) が関係し、メーカーは把握できないものも多い。この記事では、(1) 判明している情報から顧客の「隠れパラメータ」を推定し、(2) それに基づいて定期交換部品の寿命を推定し、(3) 最適な交換時期を決めるというテーマを取り扱う。例題として、オンデマンド印刷に用いるデジタル印刷機を考え、各テーマについてコンピュータシミュレーションを設計し、Python言語によって実装する。
